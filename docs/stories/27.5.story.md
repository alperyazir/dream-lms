# Story 27.5: Production Deployment Pipeline

## Status

Draft

## Story

**As a** platform administrator,
**I want** a repeatable production deployment pipeline with health checks, backups, and rollback,
**so that** I can safely deploy and operate Dream LMS on Hetzner + Cloudflare.

## Acceptance Criteria

1. `docker compose -f docker-compose.prod.yml up` starts the full production stack
2. GitHub Actions pipeline: push to main triggers lint + test + build
3. Production deploy via single command or CI trigger
4. SSL/TLS configured and verified (Traefik + Let's Encrypt or Cloudflare)
5. Health check endpoint (`/health`) returns system status (DB, Redis, DCS connectivity)
6. Automated daily PostgreSQL backups with retention policy
7. Rollback procedure tested and documented
8. Deployment runbook written in `docs/deployment-runbook.md`

## Tasks / Subtasks

- [ ] Task 1: Optimize Docker production images (AC: 1)
  - [ ] Backend Dockerfile: multi-stage build, minimal base image, non-root user
  - [ ] Frontend Dockerfile: build stage + nginx serve stage
  - [ ] Verify image sizes are reasonable (<500MB each)
  - [ ] Add `.dockerignore` to exclude `node_modules`, `venv`, `.git`, tests

- [ ] Task 2: Create production Docker Compose (AC: 1)
  - [ ] Create `docker-compose.prod.yml` with all services
  - [ ] Configure resource limits (memory, CPU) per service
  - [ ] Add restart policies (`unless-stopped`)
  - [ ] Add container health checks for each service
  - [ ] Configure logging driver (json-file with rotation)
  - [ ] Production environment variables via `.env.prod` template

- [ ] Task 3: Build CI/CD pipeline (AC: 2, 3)
  - [ ] Create `.github/workflows/ci.yml` — lint + test on PR
  - [ ] Create `.github/workflows/deploy.yml` — build + push + deploy on main
  - [ ] Add branch protection rules documentation
  - [ ] Configure Docker image registry (GitHub Container Registry or Docker Hub)
  - [ ] SSH deploy step to Hetzner VPS (or docker context)

- [ ] Task 4: Configure SSL/TLS (AC: 4)
  - [ ] Traefik configuration with Let's Encrypt auto-renewal
  - [ ] Or: Cloudflare SSL with origin certificate
  - [ ] Verify HTTPS redirect works
  - [ ] Test certificate auto-renewal

- [ ] Task 5: Build health check endpoint (AC: 5)
  - [ ] Create `/health` endpoint that checks: DB connection, Redis connection, DCS reachability
  - [ ] Return structured status response with component health
  - [ ] Add Docker health check that calls this endpoint
  - [ ] Configure uptime monitoring (Uptime Kuma or similar)

- [ ] Task 6: Set up database backups (AC: 6)
  - [ ] Create backup script using `pg_dump`
  - [ ] Configure cron job for daily backups
  - [ ] Set retention policy (keep last 7 daily, 4 weekly)
  - [ ] Test backup restoration procedure
  - [ ] Document backup/restore in runbook

- [ ] Task 7: Document rollback procedure (AC: 7)
  - [ ] Document how to rollback Docker images to previous version
  - [ ] Document Alembic migration rollback (`alembic downgrade -1`)
  - [ ] Test rollback on staging/local environment
  - [ ] Include rollback steps in deployment runbook

- [ ] Task 8: Write deployment runbook (AC: 8)
  - [ ] First-time server setup (Docker, dependencies, DNS)
  - [ ] Standard deployment procedure (step by step)
  - [ ] Rollback procedure
  - [ ] Troubleshooting common issues
  - [ ] Environment variable reference
  - [ ] Monitoring dashboard links
  - [ ] Save to `docs/deployment-runbook.md`

## Dev Notes

### Existing Infrastructure

- **Dockerfiles exist:** `backend/Dockerfile`, `frontend/Dockerfile`
- **Docker Compose exists:** `docker-compose.yml` (development)
- **PgBouncer config:** `pgbouncer/pgbouncer.ini` (already production-tuned)
- **PostgreSQL tuning:** `postgresql-tuning.conf` (4GB RAM config)
- **Traefik:** Referenced in tech stack but may not be configured yet
- **Deployment target:** Hetzner VPS, Cloudflare DNS/CDN

### Service Architecture

```
Cloudflare (DNS + CDN)
  └── Traefik (reverse proxy + SSL)
       ├── Frontend (nginx)
       ├── Backend (uvicorn/gunicorn)
       ├── PgBouncer → PostgreSQL
       ├── Redis
       └── Arq Worker
```

### Environment Variables for Production

Key variables that need production values:
- `SECRET_KEY`, `FIRST_SUPERUSER_PASSWORD`
- `POSTGRES_*` (DB credentials)
- `DREAM_CENTRAL_STORAGE_API_KEY`
- `DEEPSEEK_API_KEY`, `GEMINI_API_KEY`
- `CORS_ORIGINS` (production domains)
- `ENVIRONMENT=production`

### Testing

- **Backend tests:** `cd backend && pytest`
- **Frontend build:** `cd frontend && npm run build`
- **Docker build test:** `docker compose -f docker-compose.prod.yml build`
- **Health check test:** `curl http://localhost:8000/health`
- **Backup test:** Run backup, drop test table, restore, verify
