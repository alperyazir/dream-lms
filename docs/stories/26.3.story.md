# Story 26.3: Admin Dashboard Frontend — Analytics Sections

## Status: Done

## Story

**As an** admin,
**I want** the admin dashboard to display platform usage trends, assignment performance, AI generation stats, and a link to the monitoring dashboard,
**so that** I can visualize how the platform is performing and being used without leaving the admin interface.

## Acceptance Criteria

1. **Platform Usage Section** displays:
   - Three stat cards showing DAU, WAU, MAU (total across students + teachers)
   - A line chart showing login trends over the selected period (daily granularity)
   - A period selector (7d / 30d / 90d) that filters all sections
2. **Assignment Metrics Section** displays:
   - Stat cards for total assignments, completion rate (%), and average score
   - A bar chart showing completion rate by activity type
   - A trend line for assignments created over time
3. **AI Generation Stats Section** displays:
   - A stat card for total AI generations with the most frequent type displayed
   - A donut/pie chart showing breakdown by activity type
   - A bar chart showing generation volume over time
4. **System Health Section** displays:
   - A card with a prominent link/button to open Uptime Kuma at `http://localhost:3001` (or configurable URL)
   - Brief text explaining what Uptime Kuma monitors
5. All sections handle loading states (skeleton/spinner) and error states (error message with retry) gracefully
6. Data auto-refreshes every 60 seconds using TanStack Query `refetchInterval`
7. Layout is responsive — works on tablet (2 columns) and desktop (4 columns for stat cards)
8. Existing admin dashboard stat cards ("System Overview" and "Content & Activity" sections) remain untouched

## Dev Notes

**Existing Admin Dashboard (`frontend/src/routes/_layout/admin/dashboard.tsx`):**
- Uses `PageContainer` and `PageHeader` from `@/components/Common/PageContainer` for layout
- Uses `StatCard` from `@/components/dashboard/StatCard` — supports `icon`, `label`, `value`, `trend` (optional with `value` and `direction`) [Source: frontend/src/components/dashboard/StatCard.tsx]
- TanStack Query pattern: `useQuery({ queryKey, queryFn, staleTime })` [Source: admin/dashboard.tsx:30-40]
- API calls use auto-generated `AdminService` from `@/client` — new endpoints will need to be added to the generated client OR called manually via fetch/axios
- Grid layout: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4` for stat cards
- Section headings: `<h2 className="text-xl font-semibold text-foreground mb-4">`

**Recharts Patterns (from `ProgressScoreChart.tsx`):**
- Import from `recharts`: `CartesianGrid`, `Line`, `LineChart`, `ResponsiveContainer`, `Tooltip`, `XAxis`, `YAxis`
- Wrap chart in `<ResponsiveContainer width="100%" height={250}>` for responsive sizing
- Chart sits inside a shadcn `Card` > `CardContent`
- Use `React.memo()` for chart components to avoid unnecessary re-renders
- Color scheme: teal-500 (`#14b8a6`) for primary data, cyan-500 for secondary

**StatCard Component Interface:**
```typescript
interface StatCardProps {
  icon: React.ReactNode       // Lucide icon wrapped in JSX
  label: string               // Card label text
  value: string | number      // Main metric value
  trend?: {                   // Optional trend indicator
    value: number             // Percentage change
    direction: "up" | "down"  // Arrow direction
  }
}
```
[Source: frontend/src/components/dashboard/StatCard.tsx:4-12]

**API Client Pattern:**
- Generated client at `@/client` using OpenAPI codegen — after backend endpoints are deployed, regenerate with `npm run generate-client`
- Alternatively, use direct fetch with the auth token from the auth store for new endpoints before codegen runs

**shadcn UI Components Available:**
- `Card`, `CardContent`, `CardHeader`, `CardTitle` — for section containers
- `Button` — for links/actions
- `ToggleGroup`, `ToggleGroupItem` — for period selector (if available, otherwise use Button group)
- All imported from `@/components/ui/`

**Styling Conventions:**
- Tailwind utility classes [Source: docs/architecture/tech-stack.md]
- Shadow: `shadow-neuro` (custom class used in StatCard and other cards)
- Border colors: `border-teal-100 dark:border-teal-900`
- Icon container: `bg-gradient-to-br from-teal-500 to-cyan-500`
- Dark mode support via `dark:` prefix classes
