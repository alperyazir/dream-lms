# Story 1.1: Project Foundation & Development Environment

## Status
**Approved**

## Story

**As a** developer,
**I want** a fully configured development environment with project structure, Docker setup, and CI/CD pipeline,
**so that** the team can begin feature development with consistent tooling and automated deployments.

## Acceptance Criteria

1. Monorepo structure is created with `/backend`, `/frontend`, `/docker`, and `/docs` directories
2. Backend FastAPI application runs on `localhost:8000` with a health check endpoint (`GET /api/health`) returning `{"status": "healthy"}`
3. Frontend React + Vite application runs on `localhost:5173` with a placeholder home page
4. Docker Compose configuration includes services for PostgreSQL, FastAPI backend, and React dev server
5. All services start successfully with `docker-compose up` command
6. Environment variable configuration is documented with `.env.example` files for both backend and frontend
7. README.md includes setup instructions, architecture overview, and development commands
8. GitHub Actions workflow runs on pull requests to execute linting checks (Black, Flake8 for backend; ESLint for frontend)
9. Backend includes basic project structure: routers, services, models, database, and config directories
10. Frontend includes basic structure: components, pages, hooks, services, and utils directories
11. Git repository is initialized with appropriate `.gitignore` files for Python and Node.js

### Docker Compose Configuration
[Source: architecture/8-deployment-architecture.md#81-docker-compose-configuration]

**Services to configure:**
1. **PostgreSQL**:
   - Image: `postgres:15-alpine`
   - Port: 5432
   - Environment: POSTGRES_DB=dreamlms, POSTGRES_USER=postgres, POSTGRES_PASSWORD=password
   - Volume: `postgres_data:/var/lib/postgresql/data`
   - Health check: `pg_isready`

2. **Backend**:
   - Build: `./backend/Dockerfile`
   - Port: 8000
   - Environment: DATABASE_URL (PostgreSQL connection)
   - Depends on: PostgreSQL (with health check)
   - Volume mount: `./backend:/app` for hot reload

3. **Frontend**:
   - Build: `./frontend/Dockerfile`
   - Port: 5173
   - Environment: VITE_API_BASE_URL
   - Depends on: backend
   - Volume mount: `./frontend:/app` with node_modules exclusion

**NOTE:** Nginx service is for production deployment. For MVP development, frontend dev server proxies to backend directly via Vite config.

### FastAPI Configuration Requirements
[Source: architecture/1-system-overview-architecture-diagram.md#132-backend-api-fastapi-application]

**Main Application (`app/main.py`):**
- Create FastAPI instance with title="Dream LMS API", version="0.1.0"
- Add CORS middleware with origins from settings (allow http://localhost:5173)
- Include health check endpoint at `/api/health`
- Automatic OpenAPI docs at `/docs` (FastAPI default)

**Settings (`app/core/config.py`):**
- Use `pydantic-settings` BaseSettings
- Load from `.env` file
- Include: app_name, app_version, database_url, jwt_secret_key, jwt_algorithm, cors_origins
- Use `@lru_cache` decorator on `get_settings()` for singleton pattern

### Frontend Configuration Requirements
[Source: architecture/5-frontend-architecture.md#51-project-structure]

**Vite Configuration (`vite.config.ts`):**
- Path alias: `@` → `./src`
- Dev server port: 5173
- Proxy `/api` to `http://localhost:8000` for backend communication

**Tailwind CSS:**
- Install: `tailwindcss`, `postcss`, `autoprefixer`
- Run: `npx tailwindcss init -p`
- Configure `content` paths: `["./index.html", "./src/**/*.{js,ts,jsx,tsx}"]`

**Shadcn UI:**
- Run: `npx shadcn-ui@latest init`
- Choose: Default style, Slate color, CSS variables: yes
- Components will be added in Story 1.4+ as needed

### CI/CD Requirements
[Source: PRD Epic 1, Story 1.1, AC 8]

**GitHub Actions Workflow:**
- Trigger: Pull requests and pushes to `main` and `develop` branches
- Backend linting job:
  - Python 3.11
  - Install dependencies: `pip install -e ".[dev]"`
  - Run Black: `black --check .`
  - Run Flake8: `flake8 app/ tests/`
- Frontend linting job:
  - Node.js 20
  - Install: `npm ci`
  - Run ESLint: `npm run lint`

**Linting Configuration:**
- Black: line-length=100, target-version=['py311']
- Flake8: max-line-length=100, ignore=E203,W503, exclude=.git,__pycache__,.venv,alembic
- ESLint: Use defaults from Vite React template

### Health Check Endpoint Specification
[Source: PRD Epic 1, Story 1.1, AC 2]

**Endpoint:** `GET /api/health`

**Response:**
```json
{
  "status": "healthy",
  "service": "dream-lms-api",
  "version": "0.1.0"
}
```

**Implementation:**
```python
@app.get("/api/health")
async def health_check():
    return {
        "status": "healthy",
        "service": settings.app_name,
        "version": settings.app_version
    }
```

### Important Notes

1. **Monorepo Structure**: All code lives in single repository with backend and frontend as subdirectories
   [Source: architecture/1-system-overview-architecture-diagram.md#11-architecture-philosophy]

2. **Async FastAPI**: Use async/await pattern for all endpoints (preparation for database operations in Story 1.2)
   [Source: architecture/1-system-overview-architecture-diagram.md#132-backend-api-fastapi-application]

3. **Type Safety**: Frontend uses TypeScript strict mode, backend uses Python type hints
   [Source: architecture/1-system-overview-architecture-diagram.md#15-technology-stack-summary]

4. **Docker Development**: All services run via Docker Compose, even for local development
   [Source: architecture/8-deployment-architecture.md]

5. **No External Integrations Yet**: Story 1.1 is pure foundation. Dream Central Storage integration comes in Story 2.1
   [Source: PRD Epic 2, Story 2.1]

### Testing

[Source: architecture/10-testing-strategy.md]

**Backend Testing (pytest):**
- Framework: `pytest 8.2+`, `pytest-asyncio 0.23+`
- Test file location: `backend/tests/`
- Naming convention: `test_*.py`
- Health check test:
  ```python
  import pytest
  from httpx import AsyncClient

  @pytest.mark.asyncio
  async def test_health_check(client: AsyncClient):
      response = await client.get("/api/health")
      assert response.status_code == 200
      assert response.json()["status"] == "healthy"
  ```

**Frontend Testing (Vitest):**
- Framework: `vitest`, `@testing-library/react`
- Test file location: `frontend/tests/` or co-located with components
- Naming convention: `*.test.tsx`
- Component test:
  ```typescript
  import { render, screen } from '@testing-library/react';
  import App from '@/App';

  describe('App', () => {
    it('renders placeholder home page', () => {
      render(<App />);
      expect(screen.getByText(/Dream LMS/i)).toBeInTheDocument();
    });
  });
  ```

**Testing Requirements for Story 1.1:**
- Backend: Test health endpoint returns correct response
- Frontend: Test placeholder page renders without errors
- Integration: Test Docker Compose starts all services
- CI: Test linting passes locally before committing

**Test Fixtures Setup:**
- Create `backend/tests/conftest.py` with FastAPI test client fixture
- Create `frontend/vitest.config.ts` with @testing-library setup
- Both test suites should run via: `pytest` (backend) and `npm run test` (frontend)
