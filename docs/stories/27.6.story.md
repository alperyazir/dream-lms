# Story 27.6: Final Sign-Off & Go-Live Checklist

## Status

Draft

## Story

**As a** platform administrator,
**I want** a final end-to-end verification in a production-like environment,
**so that** I can confidently launch Dream LMS knowing all critical flows work.

## Acceptance Criteria

1. All 4 user roles tested end-to-end (admin, supervisor, teacher, student)
2. AI generation works: quota enforcement, save to library, assign to students
3. Book content loads: covers, modules, activities from DCS
4. Assignment flow works: create, assign to class, student submit, teacher grade
5. Load test passes at 100+ concurrent users without errors
6. Monitoring dashboard shows real-time metrics
7. Go-live checklist completed and signed off

## Tasks / Subtasks

- [ ] Task 1: End-to-end role testing (AC: 1)
  - [ ] Admin: login, manage users, view AI usage dashboard, manage settings
  - [ ] Supervisor: login, view teachers, view classrooms, view reports
  - [ ] Teacher: login, view books, generate AI content, create assignments, grade
  - [ ] Student: login, view assignments, play activities, submit, view progress

- [ ] Task 2: AI generation flow (AC: 2)
  - [ ] Generate content with quota check (verify counter decrements)
  - [ ] Save generated content to library
  - [ ] Create assignment from library content
  - [ ] Verify quota enforcement when limit reached (button disabled, API returns 429)

- [ ] Task 3: DCS content flow (AC: 3)
  - [ ] Book list loads with covers
  - [ ] Book modules and activities load
  - [ ] Audio playback works (TTS and pre-generated)
  - [ ] Teacher materials upload and download

- [ ] Task 4: Assignment flow (AC: 4)
  - [ ] Teacher creates assignment from AI content
  - [ ] Teacher assigns to a class
  - [ ] Student sees assignment in dashboard
  - [ ] Student plays activity and submits
  - [ ] Teacher sees submission and grades it
  - [ ] Student sees grade and progress update

- [ ] Task 5: Load test (AC: 5)
  - [ ] Configure load test for 100+ concurrent users
  - [ ] Run against production-like environment
  - [ ] Verify no 500 errors under load
  - [ ] Check response times stay under 2s for page loads
  - [ ] Verify database connections don't exhaust

- [ ] Task 6: Monitoring verification (AC: 6)
  - [ ] Health check endpoint responds correctly
  - [ ] Uptime monitoring alerts on downtime
  - [ ] Error logs are captured and accessible
  - [ ] Database backup ran successfully

- [ ] Task 7: Go-live checklist (AC: 7)
  - [ ] DNS configured and resolving
  - [ ] SSL certificate valid and auto-renewing
  - [ ] Production environment variables set
  - [ ] Database migrated to latest
  - [ ] Admin superuser account configured
  - [ ] CORS restricted to production domains
  - [ ] Rate limiting active
  - [ ] Backups running
  - [ ] Monitoring active
  - [ ] Rollback procedure documented and tested
  - [ ] Stakeholder sign-off obtained
