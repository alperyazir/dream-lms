# Story 14.3: Frontend - Supervisor Management UI

**Epic:** 14 - Supervisor Role
**Story ID:** 14.3
**Status:** Done
**Created:** 2025-12-19
**Completed:** 2025-12-20
**Priority:** High
**Depends On:** Story 14.1, Story 14.2

---

## Story

**As an** Admin or Supervisor user,
**I want** a user interface to manage Supervisor users and see the Supervisor role reflected throughout the app,
**so that** I can effectively manage Supervisors and recognize their role in the system.

---

## Acceptance Criteria

### Navigation & Routing
1. "Supervisors" tab visible in Admin left navigation
2. Tab visible to both Admin and Supervisor users
3. Route `/admin/supervisors` loads Supervisors management page
4. Route `/admin/supervisors/:id` loads Supervisor detail page (optional)

### Supervisors List Page
5. Displays table of all Supervisors
6. Table columns: Name, Email, Username, Status, Created, Actions
7. Supports search by name or email
8. Supports pagination
9. Shows "No supervisors found" when empty

### Create Supervisor (Admin Only)
10. "Add Supervisor" button visible only to Admin users
11. Opens Create Supervisor dialog
12. Form fields: Full Name*, Email*, Username*
13. Form validation matches backend requirements
14. On success, shows initial password with "Copy to Clipboard"
15. Refreshes list after creation
16. Shows error toast on failure

### Edit Supervisor (Admin Only)
17. "Edit" action in row dropdown (Admin only)
18. Opens Edit Supervisor dialog with pre-filled data
19. Editable fields: Full Name, Email, Username, Active status
20. Saves changes on submit
21. Shows success/error toast

### Delete Supervisor (Admin Only)
22. "Delete" action in row dropdown (Admin only)
23. Shows confirmation dialog: "Are you sure you want to delete this supervisor?"
24. Deletes on confirm
25. Refreshes list after deletion
26. Shows success/error toast

### Password Reset (Admin Only)
27. "Reset Password" action in row dropdown (Admin only)
28. Shows confirmation dialog
29. On success, displays new password with "Copy to Clipboard"
30. Shows error toast on failure

### Role Badge Updates
31. Supervisor role has distinct badge color (e.g., orange or purple)
32. Badge displays correctly in user lists
33. Badge displays in header/profile area for logged-in Supervisor

### Danger Zone Removal
34. Remove "Danger Zone" / "Delete Account" section from Admin profile/settings
35. Remove from Publisher profile/settings
36. Remove from Teacher profile/settings
37. Remove from Student profile/settings
38. Remove from Supervisor profile/settings
39. No user can self-delete through UI

### Supervisor Dashboard
40. Supervisor sees Admin-like dashboard after login
41. Dashboard shows same stats/widgets as Admin
42. Greeting shows "Welcome, Supervisor" or user's name

## Technical Notes

### Supervisors List Page

```tsx
// frontend/src/routes/admin/supervisors/index.tsx
import { useState } from 'react';
import { useQuery } from '@tanstack/react-query';
import { DataTable } from '@/components/ui/data-table';
import { Input } from '@/components/ui/input';
import { Button } from '@/components/ui/button';
import { Plus, Search } from 'lucide-react';
import { useSupervisors } from '@/hooks/useSupervisors';
import { useAuth } from '@/hooks/useAuth';
import { SupervisorDialog } from './components/SupervisorDialog';
import { columns } from './components/columns';

export function SupervisorsPage() {
  const { user } = useAuth();
  const isAdmin = user?.role === 'admin';

  const [search, setSearch] = useState('');
  const [dialogOpen, setDialogOpen] = useState(false);

  const { data, isLoading } = useSupervisors({ search });

  return (
    <div className="space-y-4">
      <div className="flex justify-between items-center">
        <h1 className="text-2xl font-bold">Supervisors</h1>
        {isAdmin && (
          <Button onClick={() => setDialogOpen(true)}>
            <Plus className="h-4 w-4 mr-2" />
            Add Supervisor
          </Button>
        )}
      </div>

      <div className="flex gap-2">
        <div className="relative flex-1 max-w-sm">
          <Search className="absolute left-3 top-1/2 -translate-y-1/2 h-4 w-4 text-muted-foreground" />
          <Input
            placeholder="Search supervisors..."
            value={search}
            onChange={(e) => setSearch(e.target.value)}
            className="pl-9"
          />
        </div>
      </div>

      <DataTable
        columns={columns}
        data={data?.items ?? []}
        isLoading={isLoading}
        emptyMessage="No supervisors found"
      />

      <SupervisorDialog
        open={dialogOpen}
        onClose={() => setDialogOpen(false)}
      />
    </div>
  );
}
```

### Password Display Component

```tsx
// frontend/src/components/PasswordDisplay.tsx
import { useState } from 'react';
import { Check, Copy } from 'lucide-react';
import { Button } from '@/components/ui/button';

export function PasswordDisplay({ password }: { password: string }) {
  const [copied, setCopied] = useState(false);

  const copyToClipboard = () => {
    navigator.clipboard.writeText(password);
    setCopied(true);
    setTimeout(() => setCopied(false), 2000);
  };

  return (
    <div className="space-y-2">
      <p className="text-sm text-muted-foreground">
        Initial password (save this now - it won't be shown again):
      </p>
      <div className="flex items-center gap-2">
        <code className="flex-1 bg-muted p-2 rounded font-mono text-sm">
          {password}
        </code>
        <Button variant="outline" size="sm" onClick={copyToClipboard}>
          {copied ? <Check className="h-4 w-4" /> : <Copy className="h-4 w-4" />}
        </Button>
      </div>
    </div>
  );
}
```

### Key Implementation Decisions

1. **Single Page Architecture**: Combined list, create, edit, delete, and password reset functionality into single `supervisors.tsx` page following existing patterns from teachers.tsx and publishers.tsx

2. **Inline Query Hooks**: Used useQuery and useMutation directly in the page component rather than creating separate hook files, matching existing codebase patterns

3. **Shared Admin Dashboard**: Supervisors are redirected to `/admin/dashboard` rather than creating a separate supervisor dashboard, as they share the same functionality

4. **Teal Color Theme**: Used teal color (`text-teal-600`) for supervisor role in quick login to provide distinct visual identity

5. **Universal Settings Page**: Since all roles share the same settings page, removing Danger Zone from `settings.tsx` covers all roles (AC 34-39)
