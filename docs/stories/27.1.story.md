# Story 27.1: Security Audit

## Status

Draft

## Story

**As a** platform administrator,
**I want** a comprehensive security audit of the codebase,
**so that** we can identify and fix vulnerabilities before exposing the system to real users.

## Acceptance Criteria

1. Every API route reviewed for proper authentication and authorization guards
2. All user inputs validated at API boundary (no SQL injection, XSS, path traversal risks)
3. No secrets (API keys, passwords, tokens) in source code or log output
4. CORS configured for production domains only (not wildcard `*`)
5. Rate limits on auth, AI generation, and admin endpoints
6. Dependency scan completed — no critical CVEs in Python or npm packages
7. Findings report delivered with severity ratings (Critical/High/Medium/Low) and fix recommendations

## Tasks / Subtasks

- [ ] Task 1: Audit Authentication & Authorization (AC: 1)
  - [ ] List all API routes in `backend/app/api/routes/` and their auth requirements
  - [ ] Verify each route has appropriate auth dependency (`CurrentUser`, `require_role`, `TeacherOrHigher`)
  - [ ] Check for routes accidentally exposed without auth
  - [ ] Review cached user object in `deps.py` — verify permission checks aren't bypassed when user is loaded from Redis cache (known issue: `current_user.teacher` is None for cached users)
  - [ ] Verify role-based access: admin-only endpoints block teachers/students

- [ ] Task 2: Audit Input Validation (AC: 2)
  - [ ] Review all request body schemas (Pydantic models) for proper validation
  - [ ] Check path parameters for injection risks (especially book names, file paths)
  - [ ] Verify `_validate_asset_path` in DCS client catches path traversal (`../`)
  - [ ] Check file upload endpoints for size limits and type validation
  - [ ] Review SQL queries for parameterization (SQLModel should handle this, but verify raw queries)

- [ ] Task 3: Audit Secrets Management (AC: 3)
  - [ ] Grep codebase for hardcoded credentials, API keys, tokens
  - [ ] Review logging statements — ensure no tokens/passwords are logged
  - [ ] Verify `.env` is in `.gitignore` and not tracked
  - [ ] Check DCS API key handling in `dream_storage_client.py`
  - [ ] Review JWT secret key configuration

- [ ] Task 4: Audit CORS & Security Headers (AC: 4)
  - [ ] Review CORS configuration in `backend/app/main.py`
  - [ ] Verify allowed origins are restricted (not `*` in production)
  - [ ] Check for security headers: X-Content-Type-Options, X-Frame-Options, CSP
  - [ ] Review cookie settings (httpOnly, secure, sameSite)

- [ ] Task 5: Audit Rate Limiting (AC: 5)
  - [ ] List all endpoints and their rate limit decorators
  - [ ] Verify login/auth endpoints have strict rate limits
  - [ ] Verify AI generation endpoints have rate limits
  - [ ] Check if rate limiting uses Redis (survives restarts) or in-memory
  - [ ] Test rate limit bypass attempts (missing headers, different IPs)

- [ ] Task 6: Dependency Vulnerability Scan (AC: 6)
  - [ ] Run `pip audit` or `safety check` on Python dependencies
  - [ ] Run `npm audit` on frontend dependencies
  - [ ] Document any critical/high CVEs found
  - [ ] Recommend upgrades or alternatives for vulnerable packages

- [ ] Task 7: DCS Integration Security (AC: 1, 3)
  - [ ] Review API key storage and transmission
  - [ ] Verify webhook secret validation in webhook handler
  - [ ] Check storage endpoint auth (JWT fallback for `/storage/` routes)
  - [ ] Review presigned URL handling — are they scoped correctly?

- [ ] Task 8: Compile Findings Report (AC: 7)
  - [ ] Categorize findings by severity (Critical/High/Medium/Low)
  - [ ] Include reproduction steps for each finding
  - [ ] Recommend specific fixes
  - [ ] Save report to `docs/qa/security-audit-report.md`

## Dev Notes

### Key Files to Review

- **Auth system:** `backend/app/api/deps.py` (user loading, role checks, Redis cache)
- **CORS config:** `backend/app/main.py`
- **Rate limiting:** `backend/app/core/rate_limit.py`, decorators on routes
- **DCS client:** `backend/app/services/dream_storage_client.py` (API key + JWT auth)
- **Webhooks:** `backend/app/api/routes/webhooks.py` (secret validation)
- **All routes:** `backend/app/api/routes/*.py`
- **Config/secrets:** `backend/app/core/config.py`, `.env`

### Known Issues to Verify

- Cached users from Redis don't have ORM relationships loaded — any endpoint relying on `current_user.teacher` or `current_user.publisher` will fail for cached users
- DCS `/storage/` endpoints don't support API key auth — falls back to JWT

### Testing

- This is an audit story — output is a findings report, not code changes
- Report location: `docs/qa/security-audit-report.md`
