# Story 31.1: Application Rate Limiting

## Status: Ready for Review

## Story

As a system administrator, I need application-level rate limiting on all API endpoints so that the system is protected from brute-force attacks, abuse, and resource exhaustion while allowing legitimate users uninterrupted access.

## Acceptance Criteria

- [ ] Rate limiting middleware is active on all API endpoints with configurable per-tier limits
- [ ] Auth endpoints (login, password recovery/reset) are limited to 10 requests/minute per IP
- [ ] AI content endpoints are limited to 20 requests/minute per authenticated user
- [ ] Write endpoints (POST/PUT/DELETE) are limited to 60 requests/minute per user
- [ ] Read endpoints (GET) are limited to 200 requests/minute per user
- [ ] Admin endpoints have elevated limits of 300 requests/minute per user
- [ ] Rate limit state is stored in Redis (DB 1, separate from cache/celery)
- [ ] 429 responses include `Retry-After` header and user-friendly error message
- [ ] Rate limiting fails open when Redis is unavailable (requests allowed, warning logged)
- [ ] Frontend displays a toast notification when a 429 response is received
- [ ] Rate limit configuration is externalized via environment variables
- [ ] All rate-limited routes include `request: Request` as first parameter (slowapi requirement)

## Dev Notes

- **Library:** `slowapi` — decorator-based FastAPI rate limiter backed by Redis
- **Architecture doc:** `docs/architecture/14-rate-limiting.md` — full design details
- **Redis DB:** Use DB 1 for rate limiting (`redis://redis:6379/1`), separate from cache (DB 0) and Celery (DB 2/3)
- **Key functions:** `get_remote_address` for auth endpoints (IP-based), `get_user_id_or_ip` for authenticated endpoints (user-based with IP fallback)
- **Fail-open:** If Redis is down, log a warning and allow requests through. Do NOT block users because rate limiting is unavailable.
- **slowapi quirk:** Every rate-limited route function MUST accept `request: Request` as its first parameter — slowapi extracts the request object from it. If missing, you'll get a runtime error.
- **Sliding window:** slowapi uses `fixed-window-elastic-expiry` strategy by default — this is fine for our use case.
- Do NOT add rate limiting to health check endpoints.
- When adding `@limiter.limit()` decorators, place them AFTER the `@router` decorator.

## Technical Design

### New File: `backend/app/core/rate_limit.py`

```python
from slowapi import Limiter
from slowapi.util import get_remote_address
from starlette.requests import Request

from app.core.config import settings


def get_user_id_or_ip(request: Request) -> str:
    """Extract user ID from JWT for authenticated requests, IP for anonymous."""
    auth_header = request.headers.get("Authorization", "")
    if auth_header.startswith("Bearer "):
        try:
            from app.core.security import decode_token
            token = auth_header.split(" ")[1]
            payload = decode_token(token)
            return f"user:{payload.get('sub', get_remote_address(request))}"
        except Exception:
            pass
    return f"ip:{get_remote_address(request)}"


class RateLimits:
    AUTH = "10/minute"
    AI = "20/minute"
    WRITE = "60/minute"
    READ = "200/minute"
    ADMIN = "300/minute"
    UPLOAD = "10/minute"


limiter = Limiter(
    key_func=get_user_id_or_ip,
    default_limits=[settings.RATE_LIMIT_DEFAULT],
    storage_uri=settings.RATE_LIMIT_REDIS_URL,
    strategy="fixed-window-elastic-expiry",
    enabled=settings.RATE_LIMIT_ENABLED,
)
```

### App Registration: `backend/app/main.py`

```python
from slowapi import _rate_limit_exceeded_handler
from slowapi.errors import RateLimitExceeded
from app.core.rate_limit import limiter

app.state.limiter = limiter
app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)
```

### Route Decorator Pattern

```python
# AUTH tier — IP-based
@router.post("/login/access-token")
@limiter.limit(RateLimits.AUTH, key_func=get_remote_address)
async def login(request: Request, ...):

# AI tier — user-based
@router.post("/ai-content/generate")
@limiter.limit(RateLimits.AI)
async def generate(request: Request, ...):

# WRITE tier — user-based
@router.post("/assignments")
@limiter.limit(RateLimits.WRITE)
async def create_assignment(request: Request, ...):

# READ tier — user-based
@router.get("/assignments")
@limiter.limit(RateLimits.READ)
async def list_assignments(request: Request, ...):
```

### Frontend 429 Handler

```typescript
// Add to existing API client interceptor
if (error.response?.status === 429) {
  const retryAfter = error.response.headers['retry-after'] || 60
  toast.warning(`Too many requests. Please wait ${retryAfter} seconds.`)
}
```
